# Sky Burst 测试用例设计

## 1. 文档目的

本文档用于定义 `Sky Burst` 当前版本的测试范围、通过标准和关键用例。

这是一份测试设计文档，不再把“是否通过”直接写死在用例里。执行结果应单独记录在测试报告或自动化流水线中。

## 2. 测试对象

| 项目 | 内容 |
|------|------|
| 游戏名称 | Sky Burst |
| 类型 | Canvas 纵版飞行射击游戏 |
| 版本 | v1.0 |
| 核心入口 | `index.html` |
| 核心脚本 | `game.js` |
| 样式文件 | `style.css` |

## 3. 测试范围

本轮覆盖：

- 启动页、HUD、结束页
- 键盘控制与移动边界
- 移动端虚拟按键与拖动
- 自动射击、敌机生成、Boss、碰撞、得分、生命
- 游戏启动、结束、重开状态
- 基础渲染与性能观察

本轮不覆盖：

- 音效与背景音乐
- 存档、排行榜、联网能力
- 浏览器兼容性矩阵
- 长时间挂机稳定性测试

## 5. 执行规则

### 5.1 结果记录要求

每条用例执行时应记录：

- 执行日期
- 执行人
- 实际结果
- 是否通过
- 证据链接

证据建议包括：

- 截图
- 录屏
- DOM 断言
- Canvas 帧截图
- 控制台日志

### 5.2 通过标准

除非用例明确说明“允许人工观察”，否则默认要求满足以下任一条件：

- DOM 文本或类名发生明确变化
- 状态值满足确定区间
- 元素位置变化满足明确方向和边界约束
- 定时行为发生在允许误差范围内
- 游戏对象数量变化满足预期

### 5.3 时间误差

涉及计时的用例允许以下误差：

- 连发间隔：`0.17s +/- 0.05s`
- Boss 出现时间：`16s +/- 0.5s`
- 键盘按住移动：允许因帧率产生少量位置偏差

## 6. 核心测试用例

### 6.1 UI 模块

#### TC-UI-001 启动界面显示

| 项目 | 内容 |
|------|------|
| 前置条件 | 打开 `index.html` |
| 测试步骤 | 1. 加载页面 2. 不点击开始按钮 |
| 预期结果 | `#overlay` 可见；标题为 `准备起飞`；按钮文案为 `开始游戏`；HUD 显示 `score=0`、`health=3`、`level=1` |
| 断言重点 | overlay 不包含 `is-hidden`；`#score/#health/#level` 文本正确 |

#### TC-UI-002 HUD 实时更新

| 项目 | 内容 |
|------|------|
| 前置条件 | 已开始游戏 |
| 测试步骤 | 1. 击毁至少 1 个普通敌机 2. 发生至少 1 次玩家受击 3. 等待 Boss 出现 |
| 预期结果 | 得分增加 5；生命减少 1；Boss 出现时阶段值从 1 变为 2 |
| 断言重点 | HUD 文本变化应与游戏行为一一对应 |

#### TC-UI-003 结束界面显示

| 项目 | 内容 |
|------|------|
| 前置条件 | 玩家生命为 1，且可被再次撞击 |
| 测试步骤 | 1. 让玩家再次被敌机撞击 |
| 预期结果 | `state.running=false` 对应的界面表现出现；overlay 显示 `任务失败`；按钮文案切换为 `重新开始`；文案中包含最终得分和阶段 |
| 断言重点 | overlay 从隐藏变为显示；标题和按钮文本正确 |

| 断言重点 | 边界值稳定，不出现超边或抖动穿透 |

#### TC-CTRL-008 未开始时键盘无效

| 项目 | 内容 |
|------|------|

| 预期结果 | 玩家位置不发生变化，游戏不进入运行态 |
| 断言重点 | overlay 仍可见；HUD 数值不变化 |

### 6.3 移动端控制模块

#### TC-MOBILE-001 虚拟方向键显示

| 项目 | 内容 |
|------|------|
| 前置条件 | 页面已加载 |
| 测试步骤 | 1. 设置视口宽度为 `920px` 以下 |
| 预期结果 | `.mobile-controls` 可见，存在 4 个方向按钮 |
| 断言重点 | 响应式切换生效 |

#### TC-MOBILE-002 虚拟方向键移动

| 项目 | 内容 |
|------|------|
| 前置条件 | 移动端视口，游戏运行中 |
| 测试步骤 | 分别长按四个方向按钮 500ms 后释放 |
| 预期结果 | 玩家按对应方向移动 |
| 断言重点 | `pointerdown` 生效，`pointerup` 后停止移动 |

#### TC-MOBILE-003 按钮离开后停止移动

| 项目 | 内容 |
|------|------|
| 前置条件 | 移动端视口，游戏运行中 |
| 测试步骤 | 1. 按住方向按钮 2. 触发 `pointerleave` |
| 预期结果 | 对应方向状态被清除，玩家停止继续朝该方向移动 |
| 断言重点 | 不应出现“松手后仍持续移动” |

#### TC-MOBILE-004 触屏拖动玩家

| 项目 | 内容 |
|------|------|
| 前置条件 | 触屏输入，游戏运行中 |
| 测试步骤 | 1. 在 canvas 内持续拖动 |
| 预期结果 | 玩家跟随拖动位置移动，且仍受边界限制 |
| 断言重点 | 玩家位置与触点近似同步 |

#### TC-MOBILE-005 鼠标移动不应触发拖动控制

| 项目 | 内容 |
|------|------|
| 前置条件 | 桌面端，游戏运行中 |
| 测试步骤 | 1. 仅移动鼠标，不按键 |
| 预期结果 | 玩家不会因为 `pointermove` 自动跟随鼠标 |
| 断言重点 | `pointerType=mouse` 时应被忽略 |

### 6.4 游戏机制模块

#### TC-MECH-001 自动双发射击

| 项目 | 内容 |
|------|------|
| 前置条件 | 游戏开始，玩家存活 |
| 测试步骤 | 1. 保持静止并观察 1 秒 |
| 预期结果 | 自动持续发射双发子弹；1 秒内应出现多轮射击 |
| 断言重点 | 单次射击产生 2 发子弹；连发间隔约 `0.17s` |

#### TC-MECH-002 普通敌机生成

| 项目 | 内容 |
|------|------|
| 前置条件 | 游戏运行中 |
| 测试步骤 | 1. 观察 3 秒内敌机生成 |
| 预期结果 | 普通敌机从顶部进入，存在尺寸、速度和漂移差异 |
| 断言重点 | 敌机 `y` 初值小于 0；生成间隔会随阶段上升而缩短 |

#### TC-MECH-003 Boss 生成与阶段变化

| 项目 | 内容 |
|------|------|
| 前置条件 | 游戏连续运行 |
| 测试步骤 | 1. 等待约 16 秒 |
| 预期结果 | Boss 从顶部中央进入；Boss 出现时阶段立即加 1 |
| 断言重点 | 阶段升级发生在 Boss 生成时，不要求击杀后才升级 |

#### TC-MECH-004 子弹命中普通敌机

| 项目 | 内容 |
|------|------|
| 前置条件 | 存在普通敌机，子弹可命中 |
| 测试步骤 | 1. 使子弹命中普通敌机 |
| 预期结果 | 敌机生命减少；命中产生绿色粒子；击毁后得分加 5，敌机消失 |
| 断言重点 | 单次命中只消耗 1 发子弹；击毁才加分 |

#### TC-MECH-005 子弹命中 Boss

| 项目 | 内容 |
|------|------|
| 前置条件 | Boss 已出现 |
| 测试步骤 | 1. 连续命中 Boss 直到击毁 |
| 预期结果 | Boss 血条缩短；击毁后得分加 30，并产生金色爆炸粒子 |
| 断言重点 | Boss 血条长度随生命减少而减少 |

#### TC-MECH-006 敌机撞击玩家

| 项目 | 内容 |
|------|------|
| 前置条件 | 玩家与敌机可接触，玩家生命至少为 2 |
| 测试步骤 | 1. 让敌机撞上玩家 |
| 预期结果 | 玩家生命减 1；敌机消失；玩家触发约 0.6 秒受击闪烁 |
| 断言重点 | 受击后短时间内不应重复扣血 |

#### TC-MECH-007 受击无敌时间

| 项目 | 内容 |
|------|------|
| 前置条件 | 玩家刚发生一次碰撞 |
| 测试步骤 | 1. 在闪烁期间立刻再与敌机接触 |
| 预期结果 | 不应再次扣除生命 |
| 断言重点 | `hitFlash > 0` 期间生命保持不变 |

#### TC-MECH-008 难度递增

| 项目 | 内容 |
|------|------|
| 前置条件 | 阶段从 1 提升到 2 或更高 |
| 测试步骤 | 1. 对比阶段前后的普通敌机生成速度与生命 |
| 预期结果 | 生成间隔缩短；敌机速度更快；生命值按阶段提升 |
| 断言重点 | 敌机强度递增与阶段值一致 |

### 6.5 状态流转模块

#### TC-STATE-001 点击开始

| 项目 | 内容 |
|------|------|
| 前置条件 | 初始 overlay 可见 |
| 测试步骤 | 1. 点击 `开始游戏` |
| 预期结果 | overlay 隐藏；玩家、敌机、子弹、粒子、计分和阶段进入初始运行状态 |
| 断言重点 | HUD 重置到 `0 / 3 / 1` |

#### TC-STATE-002 游戏结束后停止推进

| 项目 | 内容 |
|------|------|
| 前置条件 | 玩家即将死亡 |
| 测试步骤 | 1. 触发最后一次受击 2. 继续观察 2 秒 |
| 预期结果 | 游戏结束后不再继续推进逻辑；不再新增子弹和敌机 |
| 断言重点 | `running=false` 后对象数量不再增长 |

#### TC-STATE-003 重新开始完全重置

| 项目 | 内容 |
|------|------|
| 前置条件 | 游戏已结束，场景中存在旧敌机、旧子弹或粒子 |
| 测试步骤 | 1. 点击 `重新开始` |
| 预期结果 | 分数、生命、阶段、计时器、按键状态全部重置；旧敌机、旧子弹、旧粒子被清空 |
| 断言重点 | 不应继承上一局残留状态 |

### 6.6 渲染与基础性能

#### TC-RENDER-001 背景与星空渲染

#### TC-RENDER-002 玩家、敌机、Boss 可辨识

|------|------|

#### TC-RENDER-003 粒子生命周期

| 项目 | 内容 |
|------|------|
| 前置条件 | 发生命中或击毁 |
| 测试步骤 | 1. 观察粒子产生到消失 |

#### TC-PERF-001 60 秒基础流畅性

| 项目 | 内容 |
|------|------|
| 前置条件 | 游戏运行中 |

#### TC-PERF-002 高阶段对象堆积观察

| 项目 | 内容 |
|------|------|
| 前置条件 | 进入高阶段 |
